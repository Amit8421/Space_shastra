# Development environment

Production and development are isolated across source, hosting, secrets, and data.

| Environment | Git branch | Vercel | Supabase | Local folder |
| --- | --- | --- | --- | --- |
| Production | `main` | Production deployment | `Space Shastra` | `C:\my interior project` |
| Development | `dev` | Preview deployment | `Space Shastra Dev` | `C:\my interior project-dev` |

The hosted development URL is generated by Vercel from the `dev` branch. Local development uses `http://localhost:3001` and connects to the isolated Supabase development project.

Start the development environment from PowerShell:

```powershell
& "C:\my interior project-dev\scripts\start-dev-environment.ps1"
```

Make and test changes only in the development folder. At the end of each approved phase:

1. Review and test the Vercel Preview deployment.
2. Review any Prisma schema changes before applying them to production.
3. Merge `dev` into `main` only after approval.
4. Verify the resulting Vercel Production deployment.
